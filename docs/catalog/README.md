# Catalogs

Catalogs are generated from private-system metadata and then sanitized.

They let the public repo show the real shape of the operating system without copying private file bodies.

- `hooks.md` — trigger inventory.
- `protocols.md` — workflow contract inventory.
- `agents.md` — agent role inventory.
- `workflows.md` — major workflow families and patterns.
- `hooks/` and `agents/` — richer public pages for selected generated items.
- `workflow-stubs/` — public-safe stubs for implementation-heavy workflows.
