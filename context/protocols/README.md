# Protocols

Protocols are the durable operating contracts that hooks, agents, and humans execute.

A strong protocol includes:

- purpose,
- required capabilities,
- inputs,
- outputs,
- phase order,
- preconditions,
- pause/debounce behavior,
- workflow state writes,
- degraded mode,
- failure recovery,
- human approval boundary.

This directory contains public-safe versions of real workflow families: morning brief, end-of-day, WBR/projections/callouts, wiki maintenance, eval routing, state files, public export, and path/workflow-state contracts.
