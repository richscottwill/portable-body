# Context Directory

`context/` is the working memory and operating-contract layer.

- `body/` — durable context organs: principles, navigation, memory, current work, task surfaces, observation.
- `protocols/` — reusable workflow contracts and failure-handling rules.
- `active/` — live state, queues, locks, run records, generated status surfaces.
- `intake/` — new learnings, session notes, draft artifacts awaiting processing.
- `config/` — runtime capability maps and other machine-readable operating config.

In a private system, `active/` and `intake/` may contain real operational data. In the public reference repo they contain only examples or README files.
